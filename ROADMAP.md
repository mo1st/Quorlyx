# Roadmap

This roadmap is intentionally public and practical. It is not a promise of dates; it is the direction for the open-source project.

## Now

- Keep the GitHub release updater reliable for self-hosted installs.
- Improve installation docs and first-run setup guidance.
- Add clearer examples for AI provider setup, Knowledge Base sources, triggers, and WooCommerce use cases.
- Keep the project free from hidden install/domain tracking.

## Next

- Add more screenshots and short GIFs for common workflows.
- Improve trigger templates for WooCommerce stores, landing pages, lead capture, and content-heavy sites.
- Add example webhook recipes for CRMs, email tools, and automation platforms.
- Improve debugging output for failed AI provider requests and frontend trigger conflicts.
- Add a small example dataset for Content Insights and behavior testing.

## Later

- Better developer docs for extending triggers, providers, goals, and exports.
- More automated tests around trigger eligibility, updater behavior, and option sanitization.
- More import/export recipes for agencies managing multiple WordPress sites.
- Accessibility and performance passes for the frontend widget and admin screens.

## Not Planned

- Hidden domain tracking.
- Required Quorlyx accounts.
- Closed feature gates in the open-source plugin.
