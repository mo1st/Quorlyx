# Roadmap

This roadmap is intentionally public and practical. It is not a promise of dates; it is the direction for the open-source project.

## Now

- Keep the GitHub release updater reliable for self-hosted installs.
- Improve installation docs and first-run setup guidance.
- Add clearer examples for AI provider setup, Knowledge Base sources, triggers, WooCommerce stores, and marketplace workflows.
- Improve the first-run setup wizard for provider keys, model selection, Knowledge Base setup, trigger testing, and health checks.
- Add better default templates for common stores, service sites, content sites, and lead-generation pages.
- Keep the project free from hidden install/domain tracking.

## Next

- Support more AI providers and model families, with clearer provider-specific setup notes.
- Improve behavior prediction quality, including calibration, confidence scoring, and better visitor pattern classification.
- Improve Content Insights recommendations so they are more actionable for posts, products, categories, landing pages, and checkout flows.
- Improve trigger templates for WooCommerce stores, marketplaces, landing pages, lead capture, and content-heavy sites.
- Add more ready-made automation recipes for cart abandonment, product comparison, high-scroll visitors, returning visitors, and exit intent.
- Add example webhook recipes for CRMs, email tools, help desks, analytics tools, and automation platforms.
- Improve debugging output for failed AI provider requests, frontend trigger conflicts, Knowledge Base gaps, and WooCommerce tracking issues.
- Add a small example dataset for Content Insights, behavior prediction, and trigger testing.

## Later

- Better developer docs for extending AI providers, triggers, goals, Knowledge Base sources, exports, and webhook actions.
- More automated tests around trigger eligibility, updater behavior, option sanitization, provider calls, and behavior scoring.
- More import/export recipes for agencies and marketplace operators managing multiple WordPress sites.
- Accessibility and performance passes for the frontend widget, admin screens, trigger engine, and Content Insights processing.
- More multilingual and localization support for chat prompts, trigger messages, admin labels, and generated content.
- More WooCommerce-specific intelligence for product bundles, coupons, product categories, abandoned checkout, and post-purchase recommendations.
- Optional integrations for popular forms, CRM tools, email marketing platforms, analytics tools, and automation platforms.
- Better privacy controls for retention, exports, conversation cleanup, and site-owner data policies.

## Not Planned

- Hidden domain tracking.
- Required Quorlyx accounts.
- Closed feature gates in the open-source plugin.
